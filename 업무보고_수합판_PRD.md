# 격주 업무보고 수합판 PRD

## 1. 제품 개요

격주 업무보고 수합판은 팀별로 한글 파일을 주고받지 않고, 웹에서 업무보고 내용을 입력하면 자동으로 하나의 보고서 형태로 모아주는 내부 업무 수합 웹앱이다.

이 제품은 문서 작성 도구가 아니다. 핵심은 반복되는 보고 업무를 더 적게 쓰고, 더 쉽게 모으고, 더 빨리 확인할 수 있게 만드는 것이다.

기존 방식이 `각자 한글 파일 작성 → 파일 제출 → 수합자 편집 → 최종 공유`였다면, 새 방식은 `각자 항목 입력 → 자동 수합 → 실시간 확인 → 필요 시 출력`이다.

## 2. 제품 목표

직원은 덜 쓰고, 수합자는 덜 고치고, 보고자는 더 빨리 본다.

### 핵심 목표

- 팀별 업무보고 내용을 웹에서 직접 입력한다.
- 지난 회차 보고 내용을 복사해 반복 작성 부담을 줄인다.
- 수합자는 팀별 제출 현황을 한눈에 확인한다.
- 입력된 내용을 자동으로 하나의 보고서 미리보기로 조립한다.
- 보고자는 최종 파일을 기다리지 않고 읽기 전용 화면에서 실시간 확인한다.
- 최종 보고서는 PDF로 출력할 수 있다.

## 3. 배경 및 문제 정의

현재 격주 업무보고는 팀별로 한글 파일을 작성해 제출하고, 수합 담당자가 파일을 모아 최종 보고서를 만드는 방식으로 운영된다.

이 방식에는 다음 문제가 있다.

- 팀별 파일 양식이 조금씩 달라질 수 있다.
- 수합자가 붙여넣기, 순서 정리, 양식 맞춤에 시간을 많이 쓴다.
- 어느 팀이 제출했는지 실시간 확인하기 어렵다.
- 최종본 작성 전까지 보고자가 내용을 확인하기 어렵다.
- 반복되는 사업 내용을 매번 다시 작성해야 한다.
- 과거 보고 내용이 파일 단위로 흩어져 재활용하기 어렵다.

## 4. 제품 원칙

### 4.1 문서 편집보다 수합 흐름을 우선한다

이 제품은 한글 문서 편집기를 대체하지 않는다. 보고 내용을 데이터로 입력받고, 정해진 형식으로 자동 조립하는 데 집중한다.

### 4.2 직원 입력 부담을 최소화한다

직원이 기존 한글 파일 작성보다 더 번거롭다고 느끼면 실패한다. 첫 버전에서 가장 중요한 기능은 지난 회차 복사와 간단한 입력 화면이다.

### 4.3 수합자는 상태를 보고 움직인다

수합자는 파일을 열어 확인하는 대신, 팀별 작성 상태와 제출 여부를 현황판에서 확인한다.

### 4.4 보고자는 완성본 전에도 본다

보고자는 최종 PDF가 나오기 전에도 읽기 전용 화면에서 현재까지 입력된 내용을 확인할 수 있어야 한다.

## 5. 사용자

### 5.1 작성자

팀별 업무보고 내용을 입력하는 직원이다.

주요 니즈:

- 지난 보고 내용을 빠르게 불러오고 싶다.
- 자기 팀 내용만 간단히 작성하고 싶다.
- 저장 여부를 신경 쓰지 않고 작성하고 싶다.
- 제출 후에도 필요한 경우 수정하고 싶다.

### 5.2 수합자

전체 팀의 보고 내용을 모아 최종 보고서를 만드는 담당자다.

주요 니즈:

- 어느 팀이 작성했는지 한눈에 보고 싶다.
- 제출하지 않은 팀을 바로 확인하고 싶다.
- 전체 보고서 형태를 실시간으로 미리 보고 싶다.
- 최종 보고서를 PDF로 출력하고 싶다.

### 5.3 보고자

작성된 업무보고를 확인하는 관리자 또는 보고 대상자다.

주요 니즈:

- 최종 파일을 기다리지 않고 현재 내용을 보고 싶다.
- 전체 내용을 읽기 전용으로 확인하고 싶다.
- 필요하면 PDF로 내려받고 싶다.

## 6. MVP 범위

첫 버전은 다음 기능만 포함한다.

### 6.1 팀 목록 관리

관리자는 보고에 참여하는 팀 목록을 관리할 수 있다.

필수 기능:

- 팀 추가
- 팀 이름 수정
- 팀 표시 순서 변경
- 팀 사용 여부 설정

초기에는 조직도 연동 없이 수동 관리한다.

### 6.2 보고 회차 생성

수합자는 격주 업무보고 회차를 생성한다.

필수 기능:

- 보고 제목 입력
- 보고 기간 입력
- 작성 마감일 입력
- 회차 상태 관리

회차 상태:

- 작성 중
- 완료

격주 기간 자동 생성은 있으면 좋지만, 첫 버전에서는 직접 입력 방식으로 시작해도 된다.

### 6.3 팀별 보고 입력

작성자는 팀별 입력 화면에서 보고 내용을 작성한다.

필수 기능:

- 팀 선택
- 업무 항목 추가
- 업무 항목 수정
- 업무 항목 삭제
- 자동 저장
- 제출 버튼

업무 항목 기본 필드:

- 제목
- 주요 내용
- 향후 계획
- 비고

첫 버전에서는 입력칸을 과도하게 늘리지 않는다. 필요한 정보가 더 있으면 비고에 작성하도록 한다.

### 6.4 지난 회차 복사

작성자는 이전 회차의 자기 팀 보고 내용을 이번 회차로 복사할 수 있다.

필수 기능:

- 직전 회차 내용 불러오기
- 복사 후 수정
- 항목별 삭제 가능

이 기능은 MVP의 핵심 기능이다. 반복 보고의 부담을 줄이지 못하면 제품의 장점이 약해진다.

### 6.5 수합 현황판

수합자는 전체 팀의 작성 상태를 확인할 수 있다.

필수 표시 정보:

- 팀명
- 작성 여부
- 제출 여부
- 업무 항목 수
- 최종 수정 시간

상태:

- 작성 전
- 작성 중
- 제출 완료
- 수정 필요
- 완료

첫 버전에서는 댓글 기능 없이 상태값으로만 관리한다.

### 6.6 보고서 미리보기

입력된 내용을 하나의 보고서처럼 볼 수 있다.

필수 기능:

- 회차 제목 표시
- 보고 기간 표시
- 팀별 내용 표시
- 팀 표시 순서 반영
- 빈 팀 또는 빈 항목 처리
- 인쇄에 적합한 화면 구성

이 화면은 수합자와 보고자가 모두 볼 수 있다.

### 6.7 PDF 출력

수합자는 보고서 미리보기 화면을 PDF로 출력할 수 있다.

필수 기능:

- 현재 회차 PDF 출력
- 보고서 제목과 기간 포함
- 팀별 내용 포함

첫 버전에서는 PDF 출력만 지원한다.

### 6.8 읽기 전용 공유 화면

보고자는 읽기 전용 링크를 통해 현재 보고 내용을 확인할 수 있다.

필수 기능:

- 내용 조회
- 팀별 내용 확인
- PDF 다운로드

첫 버전에서는 별도의 복잡한 보고자 대시보드를 만들지 않는다.

## 7. MVP 제외 범위

- 항목별 댓글

- 사진 업로드

## 8. 핵심 사용자 흐름

### 8.1 보고 회차 생성

1. 수합자가 새 보고 회차를 만든다.
2. 보고 제목, 보고 기간, 작성 마감일을 입력한다.
3. 회차 상태는 작성 중으로 시작한다.

### 8.2 팀별 작성

1. 작성자가 이번 회차 작성 화면에 접속한다.
2. 자기 팀을 선택한다.
3. 지난 회차 내용을 복사한다.
4. 이번 회차에 맞게 내용을 수정한다.
5. 필요한 업무 항목을 추가하거나 삭제한다.
6. 제출 버튼을 누른다.

### 8.3 수합 확인

1. 수합자가 현황판을 연다.
2. 팀별 작성 상태와 제출 여부를 확인한다.
3. 누락된 팀 또는 수정이 필요한 팀의 상태를 변경한다.
4. 보고서 미리보기에서 전체 내용을 확인한다.

### 8.4 보고 확인 및 출력

1. 보고자는 읽기 전용 화면에서 현재 내용을 확인한다.
2. 수합자는 최종 내용을 확인한다.
3. 수합자는 PDF로 출력한다.
4. 회차 상태를 완료로 변경한다.

## 10. 화면 구성

### 10.1 메인 현황 화면

목적:

- 이번 회차의 전체 진행 상황을 보여준다.

주요 요소:

- 현재 보고 회차
- 보고 기간
- 작성 마감일
- 전체 제출 현황
- 팀별 상태 목록

### 10.2 팀별 작성 화면

목적:

- 직원이 자기 팀의 보고 내용을 작성한다.

주요 요소:

- 팀 선택
- 지난 회차 복사 버튼
- 업무 항목 목록
- 항목 추가 버튼
- 자동 저장 상태
- 제출 버튼

### 10.3 수합 현황판

목적:

- 수합자가 팀별 작성 상태를 관리한다.

주요 요소:

- 팀명
- 작성 상태
- 제출 여부
- 항목 수
- 최종 수정 시간
- 상태 변경 버튼

### 10.4 보고서 미리보기 화면

목적:

- 입력된 내용을 최종 보고서 형태로 확인한다.

주요 요소:

- 보고 제목
- 보고 기간
- 팀별 업무 내용
- PDF 출력 버튼

### 10.5 읽기 전용 공유 화면

목적:

- 보고자가 현재 보고 내용을 실시간으로 확인한다.

주요 요소:

- 보고 제목
- 보고 기간
- 팀별 내용
- PDF 다운로드 버튼

## 12. 성공 기준

### 12.1 정량 기준

- 수합자가 팀별 한글 파일을 직접 병합하지 않아도 된다.
- 전체 팀 제출 여부를 한 화면에서 확인할 수 있다.
- 작성자는 지난 회차 복사를 통해 기존 내용을 재활용할 수 있다.
- 최종 보고서를 PDF로 출력할 수 있다.

### 12.2 정성 기준

- 직원이 기존 한글 파일 작성보다 덜 번거롭다고 느낀다.
- 수합자가 붙여넣기와 양식 정리 시간이 줄었다고 느낀다.
- 보고자가 최종본 전에도 내용을 확인할 수 있다고 느낀다.
- 첫 사용자가 별도 교육 없이 기본 작성과 제출을 할 수 있다.

## 13. 리스크 및 대응

### 13.1 직원이 웹 입력을 더 불편하게 느낄 수 있음

대응:

- 입력 필드를 최소화한다.
- 지난 회차 복사를 MVP 핵심으로 둔다.
- 자동 저장을 제공한다.
- 한 화면에서 작성이 끝나도록 한다.

### 13.2 기존 보고서 양식과 출력물이 다를 수 있음

대응:

- 첫 버전은 완전한 HWP 재현보다 읽기 좋은 PDF 출력에 집중한다.
- 실제 보고 양식과 유사한 미리보기 화면을 만든다.
- HWP 출력은 사용 검증 후 후순위로 검토한다.

### 13.3 권한 설계가 복잡해질 수 있음

대응:

- 초기에는 작성 가능, 관리, 읽기 전용 3단계로 제한한다.
- 팀장 승인과 조직도 연동은 제외한다.

### 13.4 기능이 많아져 출시가 늦어질 수 있음

대응:

- MVP 제외 범위를 명확히 유지한다.
- 입력, 복사, 수합, 미리보기, PDF에 집중한다.

격주 업무보고 수합판은 팀별 한글 파일을 모으는 부담을 줄이고, 반복 보고를 웹에서 입력, 복사, 수합, 미리보기, 출력할 수 있게 만드는 내부 업무 수합 도구다.
