# 격주 업무보고 수합판 구축계획

## 1. 구축 목표

격주 업무보고 수합판 MVP를 행정망 또는 폐쇄망 고정 IP 환경에서 사용할 수 있는 내부 웹앱으로 구축한다.

첫 구축 범위는 팀 관리, 회차 관리, 팀별 보고 입력, 지난 회차 복사, 수합 현황판, 보고서 미리보기, 브라우저 기반 PDF 출력, 읽기 전용 공유 화면이다.

## 2. 구축 전제

- 일반 인터넷 공개 배포는 하지 않는다.
- 행정망 또는 폐쇄망 고정 IP에서만 접속 가능해야 한다.
- 데이터베이스는 외부 관리형 DB를 사용하지 않고 자체 PostgreSQL을 운영한다.
- 애플리케이션과 DB는 Docker Compose로 같은 배포 단위에서 운영한다.
- 담당자별 로그인은 MVP에서 제외한다.
- 관리자는 로그인하고, 작성자와 보고자는 링크 토큰으로 접근한다.
- 링크 토큰도 허용 IP 내부에서만 유효하다.

## 3. 기술 구성

- Frontend/Backend: Next.js App Router, TypeScript
- UI: Tailwind CSS, shadcn/ui 스타일 기반 컴포넌트, lucide-react
- DB: PostgreSQL
- ORM: Prisma
- 검증: Zod
- 폼: React Hook Form
- 배포: Docker Compose
- 접근 제한: 방화벽, 리버스 프록시, Next.js Proxy

## 4. 단계별 구축 계획

### 4.1 1단계: 프로젝트 기반

- Next.js 프로젝트 초기 구성
- Tailwind CSS 설정
- Prisma 설정
- Docker Compose 구성
- 환경 변수 예시 작성
- 기본 레이아웃과 관리자 진입 화면 작성

완료 기준:

- 개발 서버가 실행된다.
- 기본 화면이 표시된다.
- Prisma 스키마가 정의되어 있다.

### 4.2 2단계: 보안 기반

- 허용 IP 대역 환경 변수 정의
- Next.js Proxy에서 IP 접근 제한 적용
- 프록시 환경에서 원본 IP 신뢰 정책 정리
- 관리자 로그인 세션 구조 작성

완료 기준:

- 허용되지 않은 IP는 화면과 API에 접근할 수 없다.
- 관리자 로그인 화면도 허용 IP에서만 보인다.

### 4.3 3단계: 관리 기능

- 팀 목록 관리
- 팀 표시 순서 관리
- 팀 사용 여부 관리
- 회차 생성과 수정
- 회차별 읽기 전용 링크 생성
- 팀별 작성 링크 생성

완료 기준:

- 관리자가 팀과 회차를 만들 수 있다.
- 회차 생성 시 팀별 보고 엔트리가 준비된다.

### 4.4 4단계: 작성 기능

- 팀별 작성 화면
- 업무 항목 추가, 수정, 삭제
- 항목 순서 관리
- 자동 저장
- 제출
- 지난 회차 복사
- 지난 업무 실적과 다음 주 계획 2단 표형 입력
- 표 삽입 UI
- 행/열 편집
- Excel 표 붙여넣기

완료 기준:

- 작성자가 팀별 링크로 자기 팀 보고를 작성할 수 있다.
- 지난 회차 내용을 복사해 이번 회차에 사용할 수 있다.
- 제출 상태가 수합 현황에 반영된다.

### 4.5 5단계: 수합 및 보고

- 관리자 현황판
- 팀별 작성 상태 표시
- 상태 변경
- 보고서 미리보기
- 읽기 전용 공유 화면

완료 기준:

- 수합자가 전체 팀의 작성/제출 상태를 확인할 수 있다.
- 보고자가 읽기 전용 링크에서 보고서를 볼 수 있다.

### 4.6 6단계: 출력 및 배포

- 인쇄 전용 CSS 작성
- PDF 저장 버튼 제공
- Docker 이미지 빌드
- PostgreSQL 볼륨 설정
- 배포 환경 변수 정리
- 백업 운영 절차 작성

완료 기준:

- 보고서 미리보기와 읽기 전용 화면에서 PDF 저장이 가능하다.
- Docker Compose로 앱과 DB를 실행할 수 있다.

4. 팀 관리
5. 회차 관리
6. 팀별 작성 화면
7. 지난 회차 복사

10. PDF 출력
11. 읽기 전용 공유 화면
12. Docker 배포 정리

## 6. 보안 구축 체크리스트

- `ALLOWED_IP_RANGES`에 행정망 또는 폐쇄망 IP 대역을 설정한다.
- `TRUSTED_PROXY_IPS`에는 실제 리버스 프록시 IP만 넣는다.
- PostgreSQL 포트는 외부에 공개하지 않는다.
- 관리자 초기 비밀번호는 첫 배포 후 즉시 변경한다.
- 팀별 작성 링크와 읽기 전용 링크는 재발급할 수 있어야 한다.
- 운영 서버의 관리 포트는 운영 담당자 IP에서만 접근한다.
- DB 백업 파일은 행정망 또는 폐쇄망 내부에만 보관한다.
