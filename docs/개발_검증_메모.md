# 개발 검증 메모

## 2026-05-07

초기 프로젝트 골격을 구성했다.

완료:

- Next.js App Router 프로젝트 파일 구성
- Tailwind CSS 설정
- Prisma 6.x 스키마 작성
- PostgreSQL Docker Compose 초안 작성
- 행정망/폐쇄망 IP 접근 제한용 Next.js Proxy 작성
- 관리자 시작 화면, 팀 관리 화면, 회차 관리 화면, 보고서 미리보기 초안 작성

검증:

- `npm install --ignore-scripts` 성공
- `npm run prisma:generate` 성공
- `npm run typecheck` 성공
- `npm exec prisma -- validate` 성공
- 자동 저장 API `PATCH /api/write/items/[itemId]` 응답 확인

추가 구현:

- 팀 관리 CRUD를 Prisma Server Actions로 연결
- 회차 생성, 수정, 상태 전환, 공유 링크 재발급 기반 구현
- 회차 생성 시 활성 팀 기준 `ReportEntry` 자동 생성
- 관리자 첫 화면에 활성 팀, 현재 회차, 제출 현황 표시 기반 추가
- 로컬 개발용 `docker-compose.dev.yml` 추가
- 초기 마이그레이션 SQL 추가
- 팀별 작성 화면 추가
- 업무 항목 추가, 수정, 삭제, 순서 변경 구현
- 지난 회차 복사와 제출 상태 전환 구현
- 회차별 수합 현황판 구현
- 실제 DB 데이터 기반 보고서 미리보기 구현
- 업무 항목 자동 저장 API 구현

- 보고서 미리보기와 PDF 출력 화면을 좌우 2단 표 형태로 변경
- 본문 안의 Markdown 표를 미리보기/PDF에서 실제 표로 렌더링하도록 구현
- 작성 화면에 표 삽입/편집 UI 추가
- 표 편집 UI에서 행/열 추가, 행/열 삭제, 셀 편집 구현
- Excel에서 복사한 셀 범위 붙여넣기 지원

현재 개발 환경 제한:

- 이 Codex 데스크톱 샌드박스에서는 Next.js가 내부 서버/빌드 워커를 생성할 때 `spawn EPERM`이 발생한다.
- 같은 이유로 `npm run dev`와 `npm run build`는 이 환경에서 완주하지 못했다.
- 코드 타입 검사와 Prisma Client 생성은 통과했으므로, 일반 로컬 터미널 또는 Docker 환경에서 dev/build를 다시 확인한다.
- 이 Codex 프로세스에서는 Docker Desktop Linux Engine 파이프 접근이 거부되어 DB 컨테이너 실행도 사용자 PowerShell에서 진행해야 한다.

다음 확인:

- 일반 PowerShell 또는 행정망 배포 후보 서버에서 `npm run dev -- --hostname 127.0.0.1 --port 3000` 실행
- Docker 환경에서 `docker compose up -d --build` 실행
- 허용 IP 대역 설정 후 403 차단 동작 확인
