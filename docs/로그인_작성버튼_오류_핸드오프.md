# 로그인 후 작성 버튼이 로그인 화면으로 튕기던 오류 핸드오프

## 최종 원인

`/my` 화면 상단의 로그아웃이 `next/link`의 GET 링크로 구현되어 있었다.

```tsx
<Link href="/logout">...</Link>
```

Next.js가 링크를 사전 요청하면서 `/logout`의 GET 핸들러가 실행되었고, `/my`에 진입하는 순간 세션이 삭제되었다. 그래서 사용자는 로그인 후 `/my`까지는 오는 것처럼 보였지만, 작성 버튼을 누르면 이미 로그아웃된 상태라 `/login`으로 리다이렉트되었다.

## 최종 수정

`src/app/my/page.tsx`

- 로그아웃을 GET 링크에서 POST form 버튼으로 변경했다.
- 페이지 렌더링 또는 프리패치만으로 로그아웃이 실행되지 않게 했다.

`src/app/logout/route.ts`

- `GET` 핸들러를 제거하고 `POST`만 처리하도록 변경했다.
- 로그아웃은 사용자가 버튼을 눌러 POST 요청을 보낼 때만 실행된다.

## 함께 조정한 내용

`APP_BASE_URL`을 `localhost` 기준으로 통일했다.

- `.env`
- `.env.local`
- `.env.example`
- `docker-compose.yml`
- `src/app/admin/cycles/links/page.tsx`

이전에는 `APP_BASE_URL=http://127.0.0.1:3000`인데 사용자는 `http://localhost:3000`으로 접속하고 있어, 작성 링크/쿠키 호스트가 섞일 가능성이 있었다.

로그인 처리는 표준 POST 라우트로 변경했다.

- `src/app/login/page.tsx`의 form action을 `/api/login`으로 변경
- `src/app/api/login/route.ts` 추가
- 로그인 성공 시 쿠키를 먼저 심은 뒤, "로그인 완료" 화면에서 사용자가 `계속`을 눌러 이동하게 변경

세션 쿠키 이름은 현재 `login_cookie_probe`로 되어 있다. 디버깅 과정에서 브라우저가 이 이름의 쿠키를 정상 저장하는 것을 확인하고 임시로 세션 쿠키 이름으로 사용했다. 추후 정리 시 더 적절한 이름으로 바꿀 수 있으나, 바꿀 경우 반드시 쿠키 저장/전송 재검증이 필요하다.

## 검증 결과

아래 흐름에서 정상 동작 확인:

1. `http://localhost:3000/login`
2. 팀 사용자 계정 로그인
3. 로그인 완료 화면에서 `계속`
4. `http://localhost:3000/my`
5. 작성 버튼 클릭
6. `/write/{teamId}/{cycleId}` 작성 화면 진입

디버그 API로 로그인 세션 확인:

```text
http://localhost:3000/api/debug/auth
```

정상 상태 예시:

```json
{
  "authenticated": true,
  "hasSessionCookie": true,
  "user": {
    "role": "team_user",
    "status": "approved",
    "teamName": "광고물관리팀"
  }
}
```

## 임시 디버그 코드

문제 추적을 위해 아래 디버그 API가 추가되어 있다.

- `src/app/api/debug/auth/route.ts`
- `src/app/api/debug/cookie-test/route.ts`
- `src/app/api/debug/cookie-check/route.ts`

운영 배포 전에는 제거하거나 관리자/개발환경에서만 접근 가능하도록 제한하는 것을 권장한다.

또한 `src/lib/auth.ts`와 `src/app/api/login/route.ts`에 인증 흐름 확인용 `console.info` 로그가 일부 남아 있다. 운영 전 로그 수준을 정리하는 것이 좋다.

```powershell
cd C:\Users\user\dev\biweekly-report-board

```

## 주의 사항

- 접속 주소는 `http://localhost:3000`으로 통일한다.
- `http://127.0.0.1:3000`과 섞어 쓰면 브라우저 쿠키 공간이 달라져 로그인 상태가 다르게 보일 수 있다.
- 로그아웃처럼 상태를 변경하는 동작은 GET 링크로 만들면 안 된다. 반드시 POST form 또는 server action 등 명시적 사용자 동작으로 처리한다.
