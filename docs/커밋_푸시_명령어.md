# 커밋 및 푸시 명령어

프로젝트 폴더에서 아래 순서로 입력합니다.

```powershell
cd C:\Users\user\dev\biweekly-report-board
git status
git add .
git status
git commit -m "업무보고 항목 유형 선택 기능 추가"
git push origin main
```

## safe.directory 오류가 날 때

Git에서 `detected dubious ownership` 오류가 나오면 아래 명령어를 한 번 실행합니다.

```powershell
git config --global --add safe.directory C:/Users/user/dev/biweekly-report-board
```

## 전체 파일이 deleted로 잡혔을 때

커밋하지 말고 먼저 staged 상태를 풉니다.

```powershell
git reset
git status
```

그다음 정상 변경 파일만 다시 추가합니다.

```powershell
git add .
git status
git commit -m "업무보고 항목 유형 선택 기능 추가"
git push origin main
```
