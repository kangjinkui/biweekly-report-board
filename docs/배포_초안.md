# 배포 초안

## 1. 운영 전제

- 행정망 또는 폐쇄망 고정 IP 서버에서만 운영한다.
- 일반 인터넷 공개 배포는 하지 않는다.
- PostgreSQL은 Docker 내부 네트워크에서만 접근한다.
- 외부 접근은 리버스 프록시 또는 서버 방화벽에서 허용 IP만 통과시킨다.

## 2. 필수 환경 변수

- `DATABASE_URL`
- `SESSION_SECRET`
- `ADMIN_EMAIL`
- `ADMIN_INITIAL_PASSWORD`
- `APP_BASE_URL`
- `ALLOWED_IP_RANGES`
- `TRUSTED_PROXY_IPS`

## 3. 초기 실행 흐름

1. 운영 서버에 소스와 `.env`를 준비한다.
2. `ALLOWED_IP_RANGES`에 행정망 또는 폐쇄망 IP 대역을 설정한다.
3. Docker Compose로 PostgreSQL과 웹앱을 실행한다.
4. Prisma 마이그레이션을 적용한다.
5. 관리자 초기 계정으로 로그인한다.
6. 팀 목록과 첫 보고 회차를 생성한다.

## 4. 로컬 개발 DB 실행

로컬 개발에서는 호스트 PC에서 Prisma 마이그레이션을 실행해야 하므로 DB 포트를 `127.0.0.1`에만 열어 둔다.

```powershell
docker compose -f docker-compose.yml -f docker-compose.dev.yml up -d db
npm.cmd run prisma:migrate -- --name init
```

## 5. 백업 원칙

- PostgreSQL 볼륨은 정기 백업한다.
- 백업 파일은 행정망 또는 폐쇄망 내부 저장소에만 보관한다.
- 복구 절차는 별도 운영 문서로 작성한다.
